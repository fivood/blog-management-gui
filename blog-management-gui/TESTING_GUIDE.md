# 博客管理 GUI - 测试指南

## ⚠️ 重要修复说明

### 模块加载错误（已解决）
- **问题**: 应用启动时出现 "Cannot find module" 错误
- **根本原因**: `bcrypt` 依赖未安装在 blog-management-gui 项目中
- **修复方案**: 
  1. 添加 `bcrypt` 到 package.json dependencies
  2. 更新 electron-vite 配置以正确外部化原生模块
- **状态**: ✅ 已修复 - 构建成功，主进程包大小从 498KB 降至 105KB

### 端口冲突问题（已解决）
- **问题**: 5173 端口被其他进程占用，导致应用反复重启
- **修复方案**: 关闭占用端口的进程，或让 Vite 自动使用其他端口
- **状态**: ✅ 已修复 - 应用现在可以正常启动

## 🚀 启动应用

### 开发模式
```bash
cd blog-management-gui
npm run dev
```

这将启动：
- Vite 开发服务器（端口 5173）
- Electron 应用（自动打开窗口）

### 生产构建
```bash
npm run build
npm run start
```

## 🐛 已知问题和限制

### 当前版本限制：
1. **图片插入到编辑器**：工具栏的"插入图片"按钮目前显示提示信息，完整功能需要实现选择模式
2. **Hugo 构建和部署**：这些功能的 UI 尚未实现（后端已完成）
3. **样式编辑器**：主题自定义功能尚未实现
4. **Hugo 项目路径**：需要在配置中设置正确的 Hugo 项目路径

### 测试注意事项：
- 首次运行时，应用会创建默认配置
- 图片上传需要有效的 Hugo 项目路径
- 文章保存到 Hugo 项目的 `content/posts/` 目录
- 图片保存到 Hugo 项目的 `static/images/` 目录
